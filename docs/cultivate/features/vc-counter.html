VCカウンター

サーバー統計情報をVC名で自動表示

💡 VCカウンターとは？ メンバー数やロール数、チャンネル数などのサーバー統計情報を VCチャンネル名に自動反映する機能です。 10分間隔で自動更新され、サーバーの状況をひと目で把握できます。

概要

VCカウンターは、指定したVCチャンネルの名前を 設定したテンプレートに従って自動更新します。 メンバー数・オンライン数・ロール数など多彩なプレースホルダーが 利用可能で、四則演算によるカスタム計算もサポートします。

機能一覧

機能 説明
グローバルカウンター サーバー全体の統計情報をVC名に表示
ロールカウンター 特定ロールを保持しているメンバー数を表示（AND/OR）
計算式カウンター 複数の値を四則演算で組み合わせた結果を表示
自動更新 10分間隔でVC名を自動更新（429回避機能付き）
日時フォーマット yyyy/mm/dd HH:MM:SS などの日時表示に対応
レート制限回避 Discord APIの429エラー発生時は30分クールダウン

プリセット機能

複数のカウンター設定（プリセット）を作成できます。 各プリセットには以下の設定項目があります。

  • 名前: プリセット名（管理用）
  • 有効/無効: カウンターの動作ON/OFF
  • 対象VC: 名前を更新するVCチャンネル
  • VC名テンプレート: VC名のフォーマット（プレースホルダー使用可）

プレースホルダー一覧

グローバルカウンター

プレースホルダー 説明
{member_count} 総メンバー数
{bot_count} Bot数
{online_count} オンラインメンバー数
{vc_count} VCチャンネル数
{role_count} ロール数
{channel_count} 全チャンネル数
{tc_count} テキストチャンネル数
{thread_count} スレッド数
{forum_count} フォーラム数
{category_count} カテゴリ数
{stage_channel_count} ステージチャンネル数
{total_thread_count} 総スレッド数（概算）
{open_thread_count} オープンスレッド数
{current_date} 現在の日時
{last_updated} 最終更新日時

ロールカウンター

モード 説明
すべて保持 (AND) 指定した全ロールを持つメンバー数をカウント
いずれか保持 (OR) 指定したロールのいずれかを持つメンバー数をカウント

日時フォーマット

{current_date} や {last_updated} では 以下のトークンを使用して日時表示をカスタマイズできます。

トークン 説明 例
yyyy 4桁年 2026
yy 2桁年 26
mm 月（2桁） 07
dd 日（2桁） 20
HH 時（24時間・2桁） 14
hh 時（12時間・2桁） 02
MM 分（2桁） 05
SS 秒（2桁） 09

計算式（四則演算）

複数のプレースホルダーを組み合わせた計算結果をVC名に表示できます。 以下のように computed タイプのプレースホルダーで使用します。 

# オンライン率（%）を表示
{ {online_count} * 100 // {member_count} }%

# メンバー - Bot = 実際のユーザー数
{ {member_count} - {bot_count} }

# TC + VC + フォーラムの合計
{ {tc_count} + {vc_count} + {forum_count} }

⚠️ 注意: 計算式内で使用するプレースホルダーは事前に解決されている必要があります。 未解決のプレースホルダーが残っているとエラーになります。

コマンド一覧

/サーバーカウンター設定
サーバーカウンター設定パネルを開きます。 プリセットの作成・編集・削除が行えます。
管理者専用

注意事項

  • VC名の更新は10分間隔で行われます（即時反映ではありません）
  • Discord APIのレート制限（429）が発生した場合、該当VCは30分間更新が停止します
  • 対象VCはBotが名前を変更できる権限を持つ必要があります
  • プレースホルダー名は {キー名} の形式で記述します
  • 計算式の結果は整数に丸められます
